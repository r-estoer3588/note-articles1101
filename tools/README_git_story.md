# Git Story Generator - MVP版

**Gitコミット履歴から「開発ストーリー記事」の骨組みを自動生成するツール**

## 🎯 これは何？

あなたのGitHubリポジトリのコミット履歴から：
- 統計データ（コミット数、変更行数、タイプ別内訳）
- 開発フェーズの転換点
- ChatGPT/Claude用のストーリー生成プロンプト

を自動生成します。

## 📦 含まれるファイル

```
tools/
├── git_story_analyzer.py      # メインスクリプト（統計分析＋プロンプト生成）
├── gpt_story_prompt.md        # 生成されたプロンプト（サンプル）
└── README_git_story.md        # このファイル
```

## 🚀 使い方（5ステップ）

### Step 1: コミット履歴を抽出

あなたのリポジトリで以下を実行：

```powershell
# PowerShellの場合
cd your-repo
git log --no-merges --date=iso --pretty=format:"%ad|%h|%s" --all | Out-File -FilePath "commits.txt" -Encoding UTF8
```

```bash
# Bash/Zshの場合
cd your-repo
git log --no-merges --date=iso --pretty=format:"%ad|%h|%s" --all > commits.txt
```

### Step 2: 詳細情報を追加（オプションだが推奨）

より詳細な分析をしたい場合、変更量も取得：

```python
# extract_detailed_commits.py を実行
python C:\Repos\extract_detailed_commits.py
```

### Step 3: 分析スクリプトを実行

```powershell
cd C:\Repos\note-articles
python tools/git_story_analyzer.py
```

出力例：

### Step 4: GPTプロンプトをコピー

`tools/gpt_story_prompt.md` を開いて、内容を全コピー。

### Step 5: ChatGPT/Claudeに投げる

コピーしたプロンプトをChatGPT/Claudeに貼り付けて、「開発ストーリー記事」の骨組みを生成。

## 💡 生成される記事の構成

プロンプトは以下の構造で記事を生成します：

1. **タイトル案（3パターン）**
   - 感情訴求型
   - 技術成長型
   - 数字訴求型

2. **序章：なぜ始めたか**
   - 課題・痛み
   - 理想の姿
   - 決断の瞬間

3. **フェーズ別の開発ストーリー**
   - 各月の主要な取り組み
   - 挑戦→挫折→突破の物語構造
   - 学びの抽出

4. **終章：今とこれから**
   - 開発を通じた変化
   - 継続している改善
   - 読者へのメッセージ

## 🔧 カスタマイズ

### キーワード追加

`find_phase_transitions()` の `keywords` リストに、
あなたのプロジェクト固有のキーワードを追加：

```python
keywords = [
    # あなたのプロジェクト用
    ('authentication', '認証実装'),
    ('api', 'API設計'),
    ('database', 'DB最適化'),
    # ...
]
```

### プロンプト調整

`generate_gpt_prompt()` の最後の部分を編集して、
出力形式をカスタマイズ可能。
