# Buffer 投稿モニタリングダッシュボード 使い方ガイド

## 📖 概要

このダッシュボードは、21 日間の Buffer 投稿キャンペーン（2025 年 11 月 24 日〜12 月 14 日）のエンゲージメント指標を記録・可視化するためのツールです。

## 📁 ファイル構成

```
monitoring/
├── dashboard.html          # メインダッシュボード（ブラウザで開く）
├── monitoring_data.csv     # データ記録用CSVファイル
└── README.md              # このファイル
```

## 🚀 クイックスタート

### 1. データの記録

毎日、以下の手順でデータを記録してください：

1. `monitoring_data.csv` を Excel またはスプレッドシートアプリで開く
2. 該当日の行に数値を入力：
   - **Followers**: 現在の総フォロワー数
   - **Followers_Change**: 前日からの増減（増加は正の数、減少は負の数）
   - **Likes**: その日の投稿に付いたいいねの合計
   - **Reposts**: リポスト数の合計
   - **Replies**: 返信数の合計
   - **Profile_Clicks**: プロフィールへのクリック数
   - **Note_PV**: note 記事のページビュー数
3. ファイルを保存

#### データ記録例

```csv
Date,Followers,Followers_Change,Likes,Reposts,Replies,Profile_Clicks,Note_PV
2025-11-24,150,15,42,8,5,23,67
2025-11-25,165,15,38,6,4,19,52
```

### 2. ダッシュボードの表示

1. `dashboard.html` をダブルクリックしてブラウザで開く
2. 「📁 CSV ファイルを選択」ボタンをクリック
3. `monitoring_data.csv` を選択
4. ダッシュボードが自動的に表示されます

## 📊 ダッシュボードの見方

### サマリーカード（上部）

8 つのカードで主要指標を一目で確認：

- **総フォロワー数**: 最新のフォロワー数と期間中の増加数
- **総いいね数**: 期間中の累計いいね数
- **総リポスト数**: 期間中の累計リポスト数
- **総返信数**: 期間中の累計返信数
- **プロフィールクリック**: プロフィールへの総クリック数
- **note PV**: note 記事への総ページビュー
- **1 日平均エンゲージメント**: (いいね+リポスト+返信)÷ 日数
- **配信日数**: データが入力されている日数

### チャート

#### 1. フォロワー成長チャート

- フォロワー数の推移を折れ線グラフで表示
- 成長カーブを視覚的に確認できます

#### 2. エンゲージメント推移チャート

- いいね、リポスト、返信の推移を表示
- どのエンゲージメントが多いか比較できます

#### 3. トラフィック分析チャート

- プロフィールクリックと note PV を棒グラフで表示
- 日別のトラフィック傾向を把握

#### 4. 日次フォロワー増減チャート

- 毎日のフォロワー増減を可視化
- 緑色：増加、赤色：減少

### 週次サマリー（下部）

Week 1、Week 2、Week 3 の 3 つのカードで週ごとの成果をまとめて表示：

- フォロワー増加数
- 総エンゲージメント
- 1 日平均エンゲージメント
- いいね・リポスト・返信の内訳

## 💡 使い方のコツ

### 効果的なモニタリング方法

#### 毎日のルーティン

1. **朝または夜に 1 回データ記録**
   - Buffer の分析画面から数値を確認
   - `monitoring_data.csv` に入力
   - ダッシュボードで確認

#### 週次レビュー（推奨）

- **Week 1 終了時（11/29）**: 初期反応の分析
  - どの投稿が反応良かったか
  - 投稿時間帯の最適化検討
- **Week 2 終了時（12/6）**: 中間評価
  - フォロワー増加ペースは目標通りか
  - エンゲージメント率の推移
- **Week 3 終了時（12/13）**: 最終評価
  - 21 日間の総括
  - 次フェーズの判断

### データ入力時の注意点

1. **フォロワー数は累計で記入**

   - ❌ 間違い: 15（増加分のみ）
   - ✅ 正しい: 165（総フォロワー数）

2. **Followers_Change は増減を記入**

   - 前日 150 人 → 今日 165 人の場合：`15`
   - 前日 165 人 → 今日 160 人の場合：`-5`

3. **0 の場合も記入**
   - データがない日は `0` と入力

## 🔧 トラブルシューティング

### ダッシュボードが表示されない

**原因**: CSV ファイルが読み込まれていない

**解決策**:

1. CSV ファイルを再度選択
2. ファイル名に日本語や特殊文字が含まれていないか確認
3. 別のブラウザで試す（Chrome、Edge、Firefox を推奨）

### グラフがおかしい

**原因**: CSV のデータ形式が正しくない

**解決策**:

1. CSV 内の数値に不要な文字（カンマ、スペースなど）が含まれていないか確認
2. 日付形式が `YYYY-MM-DD` になっているか確認
3. 列の順序が正しいか確認

### CSV が開けない

**原因**: ファイルが破損している

**解決策**:

1. 元の CSV ファイルのバックアップから復元
2. 新しい CSV ファイルを作成（フォーマットは下記参照）

## 📋 CSV フォーマット仕様

```csv
Date,Followers,Followers_Change,Likes,Reposts,Replies,Profile_Clicks,Note_PV
2025-11-24,0,0,0,0,0,0,0
```

- **Date**: YYYY-MM-DD 形式
- **すべての数値列**: 整数（小数点なし）
- **カンマ区切り**
- **ヘッダー行必須**

## 📈 期待される成果の目安

### 短期目標（21 日間）

- フォロワー数: +100〜300 人
- 1 投稿あたり平均エンゲージメント: 5〜15
- note 記事への流入: 50〜150PV/週

### 判断基準

#### ✅ 成功パターン

- フォロワーが右肩上がり
- 1 日平均エンゲージメントが 5 以上
- note 記事へのトラフィックが増加傾向

#### ⚠️ 改善が必要なパターン

- フォロワーが停滞または減少
- エンゲージメントが 1〜2 程度で低迷
- プロフィールクリックがほとんどない

## 💾 データのバックアップ

定期的に CSV ファイルをバックアップすることをおすすめします：

```
monitoring/
├── monitoring_data.csv         # 最新
├── backup/
    ├── monitoring_data_1130.csv
    ├── monitoring_data_1207.csv
    └── monitoring_data_1214.csv
```

## 🎯 次のステップ

21 日間の配信が終了したら：

1. **最終データの確認**
   - すべての日のデータが入力されているか確認
2. **スクリーンショットの保存**

   - ダッシュボードの各チャートをスクリーンショット
   - 報告書や振り返り資料に使用

3. **結果に基づく判断**
   - 成功 → Week 4〜9 のリライト・配信継続
   - 改善必要 → 戦略の見直し

## ❓ よくある質問

### Q: データは毎日入力しないとダメ？

A: はい。毎日入力することで正確なトレンドが把握できます。数日まとめて入力すると、日別の変化が分からなくなります。

### Q: ダッシュボードはオンラインでないと使えない？

A: 初回読み込み時はインターネット接続が必要ですが、一度読み込めばオフラインでも使用できます。

### Q: スマートフォンでも見れる？

A: はい。レスポンシブデザインなのでスマートフォンやタブレットでも表示できます。

### Q: 複数のキャンペーンを比較したい

A: CSV ファイルを別名で保存（例：`monitoring_data_campaign1.csv`）すれば、複数のキャンペーンを比較できます。
