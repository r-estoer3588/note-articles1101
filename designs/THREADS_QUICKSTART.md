### 投稿生成ツール
`tools/threads_post_generator.py` - 投稿を自動生成するPythonスクリプト

**使い方**:

```powershell
# 対話モードで起動（推奨）
python tools/threads_post_generator.py --interactive

# 1週間分のスケジュール生成（Phase 1: 認知拡大）
python tools/threads_post_generator.py --generate-week --phase 1

# 1週間分のスケジュール生成（Phase 2: バズ創出）
python tools/threads_post_generator.py --generate-week --phase 2

# 1週間分のスケジュール生成（Phase 3: 収益化）
python tools/threads_post_generator.py --generate-week --phase 3

# 特定タイプの投稿を生成
python tools/threads_post_generator.py --type 損失回避型 --theme "時給計算"
python tools/threads_post_generator.py --type 共感型 --theme "時間ない"
python tools/threads_post_generator.py --type リスト型 --theme "AIツール"
```

**生成される投稿タイプ**:
1. 損失回避型 - 「今動かないとヤバい」を刺し込む
2. 共感型 - 「この人も同じだ」→希望提示
3. リスト型 - 保存・RT誘発、情報価値提供
4. 挑発型 - バズ・拡散狙い（炎上覚悟、週1回まで）
5. 成功体験型 - 「自分もできるかも」希望提示 + note誘導

**出力形式**:
- CSV形式で1週間分の投稿スケジュール生成
- 日付・時間・タイプ・テーマ・本文・ステータス・分析用カラム付き

## 📅 60日間のフェーズ別戦略

### Phase 1: 認知拡大（1-20日）
- **目標**: フォロワー0→500、型の確立
- **投稿数**: 毎日3投稿 × 20日 = 60投稿
- **重点**: 損失回避型を最多（週7回）、リスト型で保存・RT獲得
- **KPI**: 平均ER 3.0%以上、フォロワー増日25人

### Phase 2: バズ創出（21-40日）
- **目標**: フォロワー500→2,000、バズ投稿3本以上
- **投稿数**: 週22投稿 × 3週 = 66投稿（深夜帯追加）
- **重点**: リスト型を「バズ狙い」に特化（RT条件付き続き）
- **KPI**: 平均ER 4.0%以上、バズ投稿3本（1万imp）、フォロワー増日75人

### Phase 3: 収益化準備（41-60日）
- **目標**: フォロワー2,000→5,000、note月間流入500人
- **投稿数**: 週22投稿 × 3週 = 66投稿
- **重点**: 週10回note誘導、日曜夜に週報告（実績投稿）
- **KPI**: 平均ER 5.0%以上、note週間流入125人、フォロワー増日150人、note売上月3万円

## 🎯 投稿設計の核心

### 7要素モデル
```
1. フック（1行目）          ← スクロール停止させる
2. 欲求/悩み可視化（2-3行）  ← 「これ俺のことだ」
3. 斬新な切り口（1-2行）     ← まだ見たことないワード/視点
4. 本文（緩急ある文体）      ← 飽きさせない
5. タイムライン誘導/ツリー誘導 ← もっと見たくなる仕掛け
6. 行動促進                 ← 小さな一歩
7. CTA（エンゲージメント）   ← 質問/RTお願い/保存して
```

### 文字数・投稿時間
- **文字数**: 150-250文字が最適
- **ゴールデンタイム**: 朝7時、昼12時、夜21時
- **深夜帯**: 23時-1時（脆弱時間帯）、深夜1時以降（夜勤層）

### 斬新な切り口（ワード例）
- 「人生時給」「時間を買う」「安定は幻想」
- 「再現性ある型」「聞き方が9割」「構造批判」

## 🚨 炎上リスク管理

### 注意すべきパターン
❌ 特定職業への侮辱  
❌ 女性蔑視・差別的表現  
❌ 自殺・犯罪の助長  

### 炎上時の対処法
1. 状況確認（5分以内）- ネガティブリプ数、RT/いいね比率
2. 判断（10分以内）- 削除 or 放置
3. 対応（30分以内）- 削除なら謝罪、放置なら静観

### 挑発型投稿のルール
- **週1回まで**
- 批判は「構造」に向ける（個人攻撃しない）
- 「俺も同じだった」で共感維持

## 📊 週次・月次レビュー

### 週次レビュー（日曜夜）
1. 今週の平均ER vs 先週比
2. バズった投稿の有無（5,000imp以上）
3. フォロワー増減
4. note流入数（UTMパラメータ）
5. ネガティブリプの有無

### 月次レビュー（月初）
1. 月間総投稿数 vs 目標
2. 平均ER vs 先月比
3. フォロワー数 vs 目標
4. note流入数 vs 目標
5. note売上（Phase 3から）

### A/Bテスト例
- 語尾: 「〜ですよね」vs「〜じゃない?」
- 数字: 時給 vs 年収 vs 人生時間
- CTA: 質問 vs RT依頼 vs 保存依頼

## 💡 クイックヒント

### 運用を楽にする工夫
1. **週末に翌週分を作成**（日曜午後2時間）
   - 先週のER分析（15分）
   - 上位投稿の共通点抽出（15分）
   - 翌週21投稿の下書き作成（1時間）
   - スプレッドシートに入力（15分）

2. **ネタ帳をNotionで管理**
   - カテゴリ: 損失回避/共感/成功体験
   - 日常で感じた「底辺あるある」をメモ

3. **自動化の範囲**
   - ✅ 投稿予約（Buffer/Hootsuite）
   - ✅ データ分析（Python）
   - ❌ 完全自動投稿（ボット感NG）
   - ❌ 自動いいね・フォロー（スパム判定）

### モチベーション維持
- 小さな成功を祝う（フォロワー+50人、初note売上）
- 「やらないこと」リスト（毎日分析しない、アンチに反応しない）
- 月初に「俺の戦い」宣言ツイート
