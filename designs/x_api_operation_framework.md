## 【序章】なぜAPIベースの分析が必要なのか

X（旧Twitter）の公式ダッシュボードは「見える化」に留まり、「なぜそうなったか」「次にどう打つか」という戦略的意思決定には不十分です。

**API分析の3大メリット**
1. **因果関係の特定**：投稿形式・時間帯・文体と反応の相関を定量化
2. **競合・トレンドの先取り**:業界動向をリアルタイムでモニタリング
3. **継続的改善サイクル**：データ駆動で「勝ちパターン」を再現・強化

これは「感覚」ではなく「証拠」に基づいた運用を可能にします。

## 第2部：両アカウント共通の分析フレームワーク

### 2.1 コア分析軸（5大ピラー）

#### ① エンゲージメント効率分析

**目的**：限られたフォロワーで最大の反応を得る

**KPI定義**
- **エンゲージメント率（ER）**：`(いいね + RT + 返信) / フォロワー数 × 100`
- **拡散係数**：`RT数 / いいね数`（高いほど拡散性あり）
- **会話誘発率**：`返信数 / (いいね + RT)`（高いほど対話型コンテンツ）

**分析手法**
1. 全投稿を「形式」でタグ付け：
   - テキストのみ / 画像1枚 / 画像複数 / 動画 / 投票 / スレッド
2. 形式別のER平均を算出 → 最高パフォーマンス形式を特定
3. 時系列でERの変動を追跡 → フォロワー増加に伴うER低下を検出

**出力例**
```
| 形式 | 投稿数 | 平均ER | 平均RT/いいね比 |
|------|--------|--------|-----------------|
| 画像1枚 | 45 | 3.2% | 0.18 |
| スレッド | 12 | 5.7% | 0.31 |
| 動画 | 8 | 4.1% | 0.25 |
```

#### ② 投稿タイミング最適化

**目的**：ターゲット層が最もアクティブな時間帯を特定

**分析手法**
1. 投稿を「時間帯」×「曜日」でマトリクス化（例：月曜7-9時、水曜20-22時）
2. 各セルの平均いいね数・RT数を算出
3. ヒートマップで可視化 → 「ゴールデンタイム」を発見

**注意点**
- フォロワーのタイムゾーンを考慮（地域分布を要確認）
- 投稿直後30分・2時間・24時間での反応推移も追跡

#### ③ コンテンツテーマ分析

**目的**：どの話題が刺さるかを定量化

**分析手法**
1. 投稿を「テーマカテゴリ」で分類：
   - AI技術 / 副業ノウハウ / 投資話 / 心理・マインド / 体験談 / 警告系
2. テーマ別のER・保存数・プロフィールクリック率を比較
3. 「反応は良いが拡散されない」vs「拡散されるが会話が生まれない」を区別

**応用**
- 高ER×高拡散のテーマ → 定期配信の柱に
- 高ER×低拡散 → 既存フォロワー向けロイヤリティコンテンツ
- 低ER×高拡散 → バイラル狙い（注意：ブランド毀損リスクあり）

#### ④ フォロワー成長分析

**目的**：質の高いフォロワーを獲得し続ける

**分析手法**
1. 日次フォロワー増減を記録
2. 特定投稿後の3日間のフォロワー増加を追跡
3. フォロワーのプロフィール（取得可能範囲）から属性を推定：
   - 言語（日本語／英語）
   - 興味キーワード（プロフィール文から抽出）
   - フォロワー/フォロー比率（インフルエンサー度）

**警告指標**
- フォロワー増加率 < ER低下率 → 「量は増えたが質が下がった」
- アンフォロー率 > 5% → コンテンツとターゲットのミスマッチ

#### ⑤ 競合・トレンドモニタリング

**目的**：市場の動きを先取りし、差別化ポイントを見つける

**分析手法**
1. **競合アカウント選定**（各3〜5アカウント）
   - AI Narrative Studio → 同テーマ（AI副業・投資系）で1万フォロワー以上
   - GETHNOTE → 「底辺脱出」「低収入向け」で5千フォロワー以上
2. **週次モニタリング**
   - 競合の投稿頻度・形式・ER推移
   - 急成長した投稿の共通パターン
3. **キーワード/ハッシュタグトレンド**
   - `#AI副業` `#FIRE` `#底辺脱出` などの投稿数・反応数推移
   - 新興ハッシュタグの早期発見

**差別化戦略**
- 競合が手薄なテーマ・形式を特定 → ブルーオーシャン攻略
- 競合の高ERコンテンツを「自分の文体」で再解釈 → 独自性確保

### 2.2 データ取得・保存の実装設計

#### 推奨技術スタック
- **言語**：Python 3.10+
- **ライブラリ**：
  - `tweepy` v4.14+（X API v2 対応）
  - `pandas` / `numpy`（データ加工）
  - `matplotlib` / `seaborn`（可視化）
  - `scikit-learn`（感情分析・クラスタリング）
- **データベース**：SQLite（初期）→ PostgreSQL（スケール時）
- **自動化**：GitHub Actions / cron（日次バッチ）

#### データスキーマ例

**tweets テーブル**
```sql
CREATE TABLE tweets (
    tweet_id TEXT PRIMARY KEY,
    account TEXT,  -- 'AI_Narrative' or 'GETHNOTE'
    created_at TIMESTAMP,
    text TEXT,
    media_type TEXT,  -- 'none', 'photo', 'video', 'poll'
    like_count INT,
    retweet_count INT,
    reply_count INT,
    quote_count INT,
    impression_count INT,
    engagement_rate FLOAT,
    theme_category TEXT,  -- 手動タグ or 自動分類
    posting_hour INT,
    posting_day TEXT,
    updated_at TIMESTAMP
);
```

**followers_daily テーブル**
```sql
CREATE TABLE followers_daily (
    account TEXT,
    date DATE,
    follower_count INT,
    following_count INT,
    tweet_count INT,
    PRIMARY KEY (account, date)
);
```

#### 日次バッチ処理フロー
1. **朝6時**：前日の全投稿メトリクスを取得・更新
2. **朝6時15分**：フォロワー数記録
3. **朝6時30分**：競合アカウントの前日投稿を取得
4. **週次（日曜22時）**：週間レポート生成・可視化

## 第3部：AI Narrative Studio 専用戦略

### 3.1 ターゲット層の解像度

**ペルソナ定義**
- **年齢**：30〜39歳

- **心理状態**：
  - 将来不安（老後資金・リストラ・AI代替）

- **情報収集行動**：
  - 平日夜20〜23時にXをチェック
  - 「実体験」「具体的数字」を信頼
  - インフルエンサーより「自分に近い人」の声を重視

### 3.2 コンテンツ戦略

#### A) 投稿形式の最適配分（月30投稿の場合）

| 形式 | 投稿数 | 目的 | 例 |
|------|--------|------|-----|
| **体験談スレッド** | 8 | 信頼構築 | 「副業で月5万稼ぐまでの3ヶ月」 |
| **データ可視化（画像）** | 6 | 説得力 | 「AI副業の収益比較グラフ」 |
| **問いかけツイート** | 5 | エンゲージメント | 「今の年収で老後資金足りる？」 |
| **Tips（箇条書き）** | 5 | 実用性 | 「副業始める前の3つの準備」 |
| **ニュース解説** | 4 | 専門性 | 「ChatGPT-5が副業に与える影響」 |
| **感情共感系** | 2 | 親近感 | 「会社辞めたい…でも怖い」 |

#### B) 文体・トーン設計

**基本方針**：「信頼できる先輩」ポジション

**DO（推奨）**
- ✅ 「〜だと思います」「〜かもしれません」（断定回避）
- ✅ 「僕も最初は不安でした」（共感先行）
- ✅ 「データで見ると〜」（根拠提示）
- ✅ 「無理にやらなくていい」（押し付けない）

**DON'T（回避）**
- ❌ 「絶対稼げる」「簡単」（誇大表現）
- ❌ 「やらない人は損」（脅し）
- ❌ 「会社員はオワコン」（ターゲット否定）
- ❌ スクショ多用のnote誘導（売り込み感）

#### C) API分析による検証項目

1. **文体別ER比較**
   - 「〜だと思います」vs「〜です」の断定度合い
   - 一人称「僕」vs「私」vs なし
2. **データ提示の有無**
   - 「具体的数字あり」vs「感覚的表現」のER差
3. **CTA（行動喚起）の効果**
   - 「リプで教えて」vs「いいねで共感」vs CTAなし

**3ヶ月目標**

## 第6部：リスク管理と注意事項

### 6.1 API利用のリスク

**アカウント凍結リスク**
- 過度な自動投稿・いいね・フォローはスパム判定
- APIでの取得は問題ないが、「投稿自動化」は慎重に

**データプライバシー**
- 他ユーザーのデータ取得・保存は規約範囲内で
- 個人を特定できる形での公開は避ける

**コスト管理**
- Basic $100/月、Pro $5,000/月 → ROI試算必須
- 「分析のための分析」にならない（行動に繋がらなければ無駄）

### 6.2 運用上の注意

**データに振り回されない**
- ERが低くても「深い共感」を得た投稿は価値あり
- 数字だけでなく「質的フィードバック」（リプ内容）も重視

**炎上リスク管理**
- 特にGETHNOTEの「挑発系」は炎上と紙一重
- APIで「ネガティブメンション急増」を検知したら即対応

**疲弊しない運用**
- 毎日分析は不要。週1レビュー、月1深堀りで十分
- 「楽しく続けられる」が最優先
