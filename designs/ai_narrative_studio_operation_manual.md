## 2. 投稿形式別テンプレート

### 2.1 体験談スレッド（週2回推奨）

**構造**:
```
1ツイート目（フック）
↓
2ツイート目（問題提起）
↓
3ツイート目（解決策・実践）
↓
4ツイート目（結果・学び）
↓
5ツイート目（CTA: noteリンク or 質問投げかけ）
```

**例**:
```
【1】副業で月5万稼ぐまでの3ヶ月を正直に話します。
僕も最初は「無理だろ…」って思ってました。

【2】当時の状況:
・年収450万
・残業月30時間
・貯金50万（将来不安で夜眠れない）
「このままじゃヤバい」って焦りだけはありました。

【3】最初にやったこと:
①ChatGPTで記事作成（1時間/日）
②ココナラで「文章添削」出品
③noteで体験談を5記事投稿
→全部無料でできることから始めました。

【4】3ヶ月後:
・ココナラ売上: 3.2万
・note売上: 1.8万
・合計: 5万円
時給換算すると800円くらいだけど、「稼げた」事実が自信になった。

【5】具体的な手順をnoteにまとめました👇
[リンク]
質問あればリプください！
```

**ポイント**:
- ✅ 数字を具体的に（年収・時間・金額）
- ✅ 「僕も最初は〜」で共感先行
- ✅ 「無料でできる」ハードル下げ
- ✅ 時給換算で現実的に見せる

## 3. 文体・言葉遣いのルール

### 3.1 基本トーン

| 要素 | ルール | 例 |
|------|--------|-----|
| 一人称 | 「僕」（親近感） | ❌私 ❌俺 |
| 語尾 | 「〜だと思います」 | ❌〜です（断定しない） |
| 共感 | 「僕も最初は〜」 | 自己開示で信頼構築 |
| 数字 | 必ず具体的に | 「3ヶ月で5万」❌「すぐ稼げる」 |
| 否定 | 「無理にやらなくていい」 | 押し付けない |

### 3.2 禁止ワード

❌ 絶対稼げる  
❌ 簡単  
❌ 誰でも100万  
❌ 会社員はオワコン  
❌ 今すぐ登録！（焦らせ過ぎ）

### 3.3 推奨ワード

✅ 〜かもしれません  
✅ データで見ると  
✅ 僕の場合は  
✅ 失敗もしました  
✅ 無理にやらなくていい

## 4. note誘導の最適化

### 4.1 誘導タイミング

**絶対ルール**: スレッド最後 or 単体ツイートの補足

**NG例**:
```
【note更新しました！】
副業で稼ぐ方法👇
[リンク]
```
→ いきなりリンクは無視される

**OK例**:
```
副業で月5万稼ぐまでの話（スレッド）
【1】〜【4】体験談
【5】具体的な手順はnoteにまとめました👇
[リンク]
質問あればリプください！
```
→ 価値提供後の自然な流れ

### 4.2 リンクの貼り方

**パターン1: 直接貼り**
```
note.com/your_account/n/abc123
```

**パターン2: 短縮URL**
```
bit.ly/your-note-article
```
→ クリック率追跡可能（Bitly使用）

**推奨**: パターン2（分析しやすい）

### 4.3 CTAの工夫

**弱いCTA**:
```
続きはnoteで
```

**強いCTA**:
```
✅ 「具体的な手順」をnoteで公開中👇
✅ 失敗談も含めて全部書きました👇
✅ 質問は記事のコメント欄でどうぞ👇
```

**ポイント**:
- 絵文字で視認性UP
- 「何が書いてあるか」を明示
- 双方向性（コメント欄）で安心感

## 5. API分析に基づく改善サイクル

### 5.1 週次レビュー（日曜夜）

**チェック項目**:
1. 今週の平均ER（目標: 2.5%以上）
2. トップ3投稿の共通点
3. ワースト3投稿の反省
4. フォロワー増減（±何人？）

**Excelテンプレート**:
```
| 日付 | 投稿形式 | テーマ | ER | いいね | RT | 返信 | メモ |
|------|---------|--------|-----|-------|-----|------|------|
```

### 5.2 月次レビュー（月初）

**確認項目**:
1. 月間総投稿数 vs 目標（30投稿）
2. 平均ER vs 先月比
3. note流入数（UTMパラメータで追跡）
4. フォロワー増加数 vs 目標

**改善アクション**:
- 高ER投稿の「型」を次月増やす
- 低ER投稿の形式を減らす or 改善
- 投稿時間の最適化（ヒートマップ参照）

### 5.3 A/Bテスト例

**テスト1: 画像あり vs なし**
- 月曜: 画像あり
- 火曜: 画像なし（同じ内容）
→ ERを比較

**テスト2: 文体変更**
- 週前半: 「〜だと思います」
- 週後半: 「〜です」
→ どちらが反応良いか

**テスト3: CTA位置**
- パターンA: スレッド最後
- パターンB: 最初のツイート
→ クリック率を比較

## 7. 炎上・リスク管理

### 7.1 炎上の予兆

**警告サイン**:
- ネガティブリプが通常の3倍以上
- RT数 > いいね数（批判的拡散）
- フォロワー外からのメンション急増

**対処法**:
1. 該当ツイートを一時削除（判断難しい場合）
2. 謝罪ツイート（誤解があれば訂正）
3. 1週間は新規投稿控える

### 7.2 避けるべきトピック

❌ 政治的主張  
❌ 特定企業・個人への批判  
❌ 「絶対稼げる」系の誇大表現  
❌ 投資の具体的銘柄推奨（金融商品取引法）

### 7.3 クレーム対応

**基本方針**: 真摯に・迅速に

**テンプレート**:
```
ご指摘ありがとうございます。
説明が不十分でした。

[誤解を招いた部分の訂正]

今後は表現に気をつけます。
貴重なご意見、感謝します🙏
```

## 8. 運用を楽にする工夫

### 8.1 投稿ストック管理

**Notionテンプレート**:
```
【データベース構造】
- タイトル
- 投稿形式（体験談/データ/問いかけ/Tips/ニュース）
- 下書き本文
- 投稿予定日
- ステータス（未作成/下書き/予約済み/投稿済み）
```

**週末に翌週分を作成**:
- 日曜午後: 7投稿分の下書き
- 平日: 微調整して投稿

### 8.2 画像素材の準備

**フォルダ構成**:
```
./assets/
├── graphs/（データグラフ）
├── quotes/（名言画像）
├── tips/（箇条書き画像）
└── thumbnails/（note用サムネ）
```

**ストック数**: 各10枚以上

### 8.3 自動化ツール活用

**推奨ツール**:
- **Buffer/Hootsuite**: 投稿予約（無料枠あり）
- **Canva**: 画像作成（テンプレート豊富）
- **Notion**: 投稿管理・アイデアストック

**注意**: 完全自動化は避ける（手動感が信頼に繋がる）
